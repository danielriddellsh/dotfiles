## graphify (scoped: orpheus repo only)

**Only applies when cwd is `/Users/dan/Github/orpheus` (or subdirectory). Ignore these rules in any other repo.**

Knowledge graph for the orpheus repo lives at `../orpheus-graphify/orpheus/graphify-out/` (sibling `orpheus-graphify` repo), generated by `../orpheus-graphify/generate.sh`.

Rules (orpheus repo only):
- For codebase questions, first run `graphify query "<question>" --out ../orpheus-graphify/orpheus/graphify-out` when `../orpheus-graphify/orpheus/graphify-out/graph.json` exists. Use `graphify path "<A>" "<B>" --out ../orpheus-graphify/orpheus/graphify-out` for relationships and `graphify explain "<concept>" --out ../orpheus-graphify/orpheus/graphify-out` for focused concepts. These return a scoped subgraph, usually much smaller than `GRAPH_REPORT.md` or raw grep.
- If `../orpheus-graphify/orpheus/graphify-out/wiki/index.md` exists, use it for broad navigation instead of raw source browsing.
- Read `../orpheus-graphify/orpheus/graphify-out/GRAPH_REPORT.md` only for broad architecture review or when query/path/explain do not surface enough context.
- After modifying code in the orpheus repo, regenerate the graph against your local working tree (captures uncommitted changes on the current branch, e.g. `feat/example`): `cd ../orpheus-graphify && ./generate.sh --incremental --local ../orpheus`. Omit `--local` only if you want the graph to reflect the submodule's remote-tracked branch instead.
